# System update & operations (§26)

Admin-controlled update checks, one-click upgrades with automatic backup, and clearer manual SSH instructions.

### Features

- **Admin → System:** Compare installed vs. latest version (GitHub Releases), optional automatic checks, and tab/sidebar update hints.
- **Upcoming release preview:** When an update is available, admins see release notes from the GitHub tag (including operator-only details in the System tab).
- **Apply update:** One-click upgrade via updater sidecar — automatic operational backup, then `update.sh` on the host (maintenance mode during apply).
- **Manual update (SSH):** Simplified “How to update” dialog with copy-to-clipboard for `scripts/update.sh`.

### Fixes

- Manual update instructions no longer require a backup checkbox in the UI (backup remains recommended before SSH upgrades).

## For operators

- **New compose service:** `docsops-updater` (internal only; Docker socket + `/opt/docsops` mount). Fresh installs get `DOCSOPS_UPDATER_URL` and `DOCSOPS_UPDATER_TOKEN` in `/etc/docsops/docsops.env`.
- **DB migration:** `20260624130000_system_update_run` — run on API startup via `prisma migrate deploy`.
- **Manual upgrade path:** `sudo /opt/docsops/scripts/update.sh v0.4.0` after a backup; existing installs without updater env vars keep SSH-only updates until env is extended and `docsops-updater` is deployed.
- **Rollback:** Restore previous `docsops.env` and bundle, re-run `update.sh` with the previous tag.
